HTTP and HTTPS protocols:

HTTP, or Hypertext Transfer Protocol, is the most popular communications protocol for sending and receiving information throughout the internet. When a user wants to reach a webpage, they must first send a request to the host server specifying which files the user wants to see. The GET request is then received by the host server which determines the files that must be POSTed back to the requestor. The POST contains all of the necessary HTML information for the usersâ€™ web browser to reconstruct the webpage. These communications happen in an instant but can be slowed down if there is a lot of media files that must be transferred back to the user.

Because the internet is an open system, sending HTTP communications with personal data is not safe. Hackers are able to intercept the HTTP messages, and since HTTP is readable by humans, they can steal personal data. To combat this security risk, HTTPS, or Hypertext Transfer Protocol Secure, was created. HTTPS uses SSL (Secure Sockets Layer) and TLS (Transport Layer Security) to encrypt the communication between the user and the host servers. So, although hackers might still intercept copies of the communication, the SSL encryption makes it extremely difficult for them to decipher the contents of the commutation. In order to promote more usage of HTTPS, Google announce in 2018 that it would begin prioritizing HTTPS sites in its search results. It also started warning users of its Chrome browser that non-HTTPS sites are insecure and unsafe.
